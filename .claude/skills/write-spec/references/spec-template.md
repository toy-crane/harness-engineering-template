## 개요
이 feature가 해결하는 문제와 사용자에게 나타나는 관찰 가능한 변화를 1-2 문장으로.

## 범위

### 포함
- 이번에 사용자가 경험할 것

### 제외
- 의도적으로 제외하는 것과 그 이유

## 시나리오

### 1. 시나리오 제목
- **Given** — 사용자나 시스템의 초기 상태
- **When** — 사용자가 취하는 행동
- **Then** — 관찰 가능한 결과

성공 기준:
- [ ] 구체적 입력 → 관찰 가능한 출력 (UI, API 응답, 저장 상태)
- [ ] 구체적 입력 → 관찰 가능한 출력

### 2. 시나리오 제목
- **Given** — ...
- **When** — ...
- **Then** — ...

성공 기준:
- [ ] ...

(3-5개 시나리오)

## 불변 규칙 (선택)

모든 시나리오에 걸쳐 유지되어야 하는 규칙. 범주 예: 보안/프라이버시, 성능, 데이터 일관성.

해당 없는 feature면 섹션을 생략한다.
