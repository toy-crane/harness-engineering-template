# Spec Test Guard

`*.spec.test.tsx` 파일은 수용 기준(acceptance criteria)을 코드로 표현한 **불변 계약**이다.

## 원칙

- spec 테스트가 실패하면, **구현 코드를 수정**하여 테스트를 통과시킨다
- spec 테스트 자체를 수정하여 실패를 우회하지 않는다

## 수정이 필요한 경우

요구사항 자체가 변경되어 spec 테스트 수정이 불가피한 경우에 한해 수정을 허용한다.
이때 반드시 다음 절차를 따른다:

1. **사용자 확인**: 수정 전 사용자에게 변경 사유와 영향 범위를 설명하고 승인을 받는다
2. **사유 기록**: 커밋 메시지에 spec 테스트 변경 사유를 명시한다

## 금지 사항

- 구현 편의를 위해 spec 테스트의 기대값을 변경하는 것
- 실패하는 spec 테스트를 `.skip`하거나 삭제하는 것
- 사용자 확인 없이 spec 테스트를 수정하는 것
