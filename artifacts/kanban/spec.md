## Overview
Trello 스타일의 칸반 보드 Todo 앱. 3개 칼럼에서 카드를 드래그&드롭으로 관리하고, 검색/필터/다크모드를 지원하며, 데이터를 localStorage에 저장한다.

## 시나리오

### 1. 칸반 보드 초기 표시
[상황] 사용자가 앱을 처음 열면
[동작] 시스템이 "To Do", "In Progress", "Done" 3개 칼럼을 표시한다

성공 기준:
- [ ] 앱 로드 시 -> 3개 칼럼("To Do", "In Progress", "Done")이 좌→우 순서로 표시됨
- [ ] 카드가 없는 초기 상태 -> 각 칼럼에 빈 상태 표시

### 2. 카드 생성
[상황] 사용자가 카드 추가 버튼을 클릭하면
[동작] 시스템이 "To Do" 칼럼에 새 카드를 생성한다

성공 기준:
- [ ] 제목 "회의록 작성" 입력 후 저장 -> "To Do" 칼럼에 "회의록 작성" 카드 표시
- [ ] 제목 없이 저장 시도 -> "제목을 입력해주세요" 에러 메시지 표시
- [ ] 제목만 입력하고 나머지 필드 비워둔 채 저장 -> 카드 정상 생성

### 3. 카드 미리보기 표시
[상황] 보드에 카드가 존재하면
[동작] 시스템이 카드에 제목, 우선순위 뱃지, 태그 칩, 마감일을 표시한다

성공 기준:
- [ ] 우선순위 "High", 태그 "버그", 마감일 "2026-03-20"인 카드 -> 제목, "High" 뱃지, "버그" 칩, "2026-03-20" 표시
- [ ] 우선순위/태그/마감일이 없는 카드 -> 제목만 표시

### 4. 카드 상세 조회 및 편집
[상황] 사용자가 카드를 클릭하면
[동작] 시스템이 모달을 열고 모든 필드를 인라인 편집할 수 있게 한다

성공 기준:
- [ ] 카드 클릭 -> 모달에 제목, 설명, 우선순위, 태그, 마감일 필드 표시
- [ ] 모달에서 제목을 "회의록 작성"에서 "주간 회의록 작성"으로 변경 후 저장 -> 보드에 "주간 회의록 작성"으로 반영
- [ ] 모달에서 우선순위를 "High"로 변경 -> 카드에 우선순위 "High" 표시

### 5. 모달 닫기 시 변경 폐기
[상황] 사용자가 모달에서 필드를 수정한 후
[동작] 저장하지 않고 모달을 닫으면 변경 사항이 폐기된다

성공 기준:
- [ ] 모달에서 제목을 "주간 회의록 작성"으로 변경 후 저장 없이 닫기 -> 보드에 원래 제목 "회의록 작성" 유지

### 6. 카드 삭제
[상황] 사용자가 카드 삭제를 실행하면
[동작] 시스템이 해당 카드를 보드에서 제거한다

성공 기준:
- [ ] 카드 삭제 실행 -> 해당 카드가 보드에서 사라짐
- [ ] 3개 카드 중 1개 삭제 -> 남은 카드 2개 표시

### 7. 칼럼 간 드래그&드롭
[상황] 사용자가 카드를 다른 칼럼으로 드래그하면
[동작] 시스템이 카드를 대상 칼럼으로 이동한다

성공 기준:
- [ ] "To Do"의 카드를 "In Progress"로 드래그 -> "In Progress" 칼럼에 카드 표시, "To Do"에서 사라짐
- [ ] "In Progress"의 카드를 "Done"으로 드래그 -> "Done" 칼럼에 카드 표시

### 8. 칼럼 내 순서 변경
[상황] 사용자가 같은 칼럼 내에서 카드를 드래그하면
[동작] 시스템이 카드 순서를 변경한다

성공 기준:
- [ ] "To Do" 칼럼에 카드 A, B, C가 있을 때 C를 A 위로 드래그 -> 순서가 C, A, B로 변경

### 9. 제목 검색
[상황] 사용자가 검색창에 키워드를 입력하면
[동작] 시스템이 제목에 키워드를 포함하는 카드만 표시한다

성공 기준:
- [ ] "회의" 검색 -> "회의록 작성" 카드만 표시, "장보기" 카드는 숨김
- [ ] 검색어 삭제 -> 모든 카드 다시 표시
- [ ] 검색 결과 없음 -> 빈 상태 메시지 표시

### 10. 우선순위 필터
[상황] 사용자가 우선순위 필터를 선택하면
[동작] 시스템이 해당 우선순위의 카드만 표시한다

성공 기준:
- [ ] "High" 필터 선택 -> 우선순위가 "High"인 카드만 표시
- [ ] 필터 해제 -> 모든 카드 표시

### 11. 태그 필터
[상황] 사용자가 태그 필터를 선택하면
[동작] 시스템이 해당 태그를 포함하는 카드만 표시한다

성공 기준:
- [ ] "버그" 태그 필터 선택 -> "버그" 태그가 있는 카드만 표시
- [ ] 우선순위 "High" + 태그 "버그" 동시 필터 -> 두 조건을 모두 만족하는 카드만 표시 (AND 조합)

### 12. 필터 해제
[상황] 사용자가 적용된 필터를 해제하면
[동작] 시스템이 모든 카드를 다시 표시한다

성공 기준:
- [ ] 우선순위 필터 해제 -> 모든 우선순위 카드 표시
- [ ] 태그 필터 해제 -> 모든 태그 카드 표시

### 13. 검색과 필터 동시 적용
[상황] 사용자가 검색어와 필터를 동시에 사용하면
[동작] 시스템이 검색어와 필터 조건을 모두 만족하는 카드만 표시한다 (AND 조합)

성공 기준:
- [ ] "회의" 검색 + "High" 우선순위 필터 -> 제목에 "회의"를 포함하고 우선순위가 "High"인 카드만 표시
- [ ] "회의" 검색 + "버그" 태그 필터 -> 제목에 "회의"를 포함하고 "버그" 태그가 있는 카드만 표시

### 14. 마감일 경과 표시
[상황] 카드의 마감일이 오늘 이전이면
[동작] 시스템이 해당 카드에 overdue 시각적 강조를 표시한다

성공 기준:
- [ ] 마감일이 어제인 카드 -> 마감일에 overdue 강조 표시
- [ ] 마감일이 내일인 카드 -> 일반 마감일 표시 (강조 없음)
- [ ] 마감일이 없는 카드 -> overdue 표시 없음

### 15. 다크모드 토글
[상황] 사용자가 다크모드 토글을 클릭하면
[동작] 시스템이 라이트/다크 테마를 전환한다

성공 기준:
- [ ] 다크모드 토글 클릭 -> 배경이 어두운 테마로 전환
- [ ] 다시 토글 클릭 -> 라이트 테마로 복원
- [ ] 다크모드 상태에서 새로고침 -> 다크모드 유지

### 16. localStorage 자동 저장
[상황] 사용자가 카드를 생성/수정/삭제/이동하면
[동작] 시스템이 변경사항을 localStorage에 자동 저장한다

성공 기준:
- [ ] 카드 생성 후 새로고침 -> 생성한 카드가 유지됨
- [ ] 카드를 다른 칼럼으로 이동 후 새로고침 -> 이동한 위치 유지

### 17. JSON 내보내기
[상황] 사용자가 내보내기를 실행하면
[동작] 시스템이 전체 보드 데이터를 JSON 파일로 다운로드한다

성공 기준:
- [ ] 내보내기 실행 -> .json 파일 다운로드됨
- [ ] 다운로드된 파일에 모든 칼럼과 카드 데이터 포함

### 18. JSON 가져오기
[상황] 사용자가 JSON 파일을 가져오면
[동작] 시스템이 기존 데이터를 덮어쓰고 가져온 데이터로 보드를 표시한다

성공 기준:
- [ ] 유효한 JSON 파일 가져오기 -> 기존 데이터 대체, 가져온 데이터로 보드 표시
- [ ] 잘못된 형식의 JSON 파일 가져오기 -> "올바른 형식의 파일이 아닙니다" 에러 메시지 표시

### 19. 태그 자동완성
[상황] 사용자가 카드에 태그를 입력하면
[동작] 시스템이 기존에 사용된 태그를 자동완성으로 제안한다

성공 기준:
- [ ] 기존에 "버그" 태그 사용 후, 새 카드에서 "버"를 입력 -> "버그" 자동완성 제안 표시
- [ ] 자동완성 목록에 없는 새 태그 입력 -> 새 태그로 저장

## 범위

### 포함
- 3칼럼 칸반 보드 (To Do, In Progress, Done)
- 카드 CRUD (제목, 설명, 우선순위, 태그, 마감일)
- 카드 미리보기: 제목, 우선순위 뱃지, 태그 칩, 마감일
- 모달 기반 카드 편집 (저장 없이 닫으면 변경 폐기)
- 칼럼 간 / 칼럼 내 드래그&드롭
- 제목 검색
- 우선순위 필터, 태그 필터 (AND 조합)
- 검색 + 필터 동시 적용 (AND 조합)
- 태그 자유 입력 + 자동완성
- 마감일 경과 시 overdue 시각적 강조
- 다크모드 토글 (상태 유지)
- localStorage 자동 저장
- JSON 내보내기/가져오기 (가져오기 시 덮어쓰기)

### 제외
- 칼럼 추가/삭제/이름 변경: 3칼럼 고정으로 MVP 범위를 축소하기 위해
- 다중 보드: 단일 보드로 시작하여 복잡도를 제한하기 위해
- 사용자 인증/로그인: 로컬 전용 앱이므로 불필요
- 실시간 협업: 로컬 전용 앱이므로 불필요
- 카드 첨부파일: localStorage 용량 제한으로 인해
- 서버 연동/API: 로컬 전용 앱이므로 불필요
