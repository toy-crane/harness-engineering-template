## Overview
Trello 스타일의 칸반 Todo 앱. 3개 고정 칼럼에서 카드를 드래그&드롭으로 관리하고, 검색/필터, 다크모드, 데이터 내보내기/가져오기를 지원한다.

## 시나리오

### 1. 카드 추가
[상황] 사용자가 Todo 칼럼에서 제목을 입력하고 카드를 추가하면
[동작] 해당 칼럼의 맨 아래에 새 카드가 생성된다 (기본 우선순위: Medium)

성공 기준:
- [ ] 제목 "회의록 작성" 입력 후 추가 -> Todo 칼럼에 "회의록 작성" 카드가 표시되고 우선순위는 Medium
- [ ] 빈 제목으로 추가 시도 -> 카드가 생성되지 않고 에러 메시지 "제목을 입력해주세요" 표시

### 2. 카드 삭제
[상황] 사용자가 카드의 삭제 버튼을 클릭하면
[동작] 확인 대화상자가 표시되고, 확인 시 카드가 삭제된다

성공 기준:
- [ ] 삭제 버튼 클릭 -> "정말 삭제하시겠습니까?" 확인 대화상자 표시
- [ ] 확인 클릭 -> 카드가 목록에서 사라짐
- [ ] 취소 클릭 -> 카드가 그대로 유지됨

### 3. 카드 인라인 편집
[상황] 사용자가 카드 제목을 클릭하면
[동작] 제목이 편집 가능한 입력 필드로 변경되고, 포커스 해제 시 저장된다

성공 기준:
- [ ] 카드 제목 "회의록 작성" 클릭 -> 입력 필드로 변경
- [ ] "주간 보고서 작성"으로 수정 후 포커스 해제 -> 제목이 "주간 보고서 작성"으로 변경

### 4. 카드 상세 편집 (모달)
[상황] 사용자가 카드를 클릭하여 상세 모달을 열면
[동작] 우선순위, 태그, 마감일, 서브태스크를 편집할 수 있다

성공 기준:
- [ ] 우선순위를 "High"로 변경 -> 카드에 High 색상 표시
- [ ] 태그 "버그" 입력 후 추가 -> 카드에 "버그" 태그 표시
- [ ] 마감일 "2026-03-20" 설정 -> 카드에 마감일 표시
- [ ] 서브태스크 "자료 조사" 추가 -> 카드에 진행률 "0/1" 표시

### 5. 서브태스크 토글
[상황] 사용자가 서브태스크의 체크박스를 토글하면
[동작] 서브태스크 완료 상태가 변경되고 진행률이 업데이트된다

성공 기준:
- [ ] 서브태스크 3개 중 2개 완료 -> 진행률 "2/3" 표시
- [ ] 모든 서브태스크 완료 -> 진행률 "3/3" 표시

### 6. 칼럼 간 드래그&드롭
[상황] 사용자가 카드를 다른 칼럼으로 드래그&드롭하면
[동작] 카드가 대상 칼럼으로 이동한다

성공 기준:
- [ ] Todo의 카드를 In Progress로 드래그 -> In Progress 칼럼에 카드 표시, Todo에서 사라짐
- [ ] In Progress의 카드를 Done으로 드래그 -> Done 칼럼에 카드 표시

### 7. 같은 칼럼 내 순서 변경
[상황] 사용자가 같은 칼럼 안에서 카드를 드래그&드롭하면
[동작] 카드 순서가 변경된다

성공 기준:
- [ ] 카드 A, B, C 순서에서 C를 A 위로 드래그 -> C, A, B 순서로 변경

### 8. 마감일 경고
[상황] 카드의 마감일이 현재 날짜보다 이전이면
[동작] 카드에 시각적 경고가 표시된다 (빨간 테두리)

성공 기준:
- [ ] 마감일이 어제인 카드 -> 빨간 테두리 표시
- [ ] 마감일이 내일인 카드 -> 빨간 테두리 없음

### 9. 카드 검색
[상황] 사용자가 검색창에 텍스트를 입력하면
[동작] 제목에 해당 텍스트가 포함된 카드만 표시된다

성공 기준:
- [ ] "회의" 검색 -> "회의록 작성" 카드만 표시, "주간 보고서" 카드는 숨겨짐
- [ ] 검색어 삭제 -> 모든 카드 다시 표시

### 10. 우선순위 필터
[상황] 사용자가 우선순위 필터를 선택하면
[동작] 해당 우선순위의 카드만 표시된다

성공 기준:
- [ ] "High" 필터 선택 -> High 우선순위 카드만 표시
- [ ] 필터 해제 -> 모든 카드 표시

### 11. 태그 필터
[상황] 사용자가 태그 필터를 선택하면
[동작] 해당 태그가 있는 카드만 표시된다

성공 기준:
- [ ] "버그" 태그 필터 선택 -> "버그" 태그가 있는 카드만 표시
- [ ] 필터 해제 -> 모든 카드 표시

### 12. 다크모드 토글
[상황] 사용자가 헤더의 다크모드 토글을 클릭하면
[동작] 라이트/다크 모드가 전환되고 선택이 저장된다

성공 기준:
- [ ] 라이트 모드에서 토글 클릭 -> 다크 모드로 전환
- [ ] 페이지 새로고침 -> 다크 모드 유지

### 13. 데이터 내보내기
[상황] 사용자가 내보내기 버튼을 클릭하면
[동작] 현재 보드 데이터가 JSON 파일로 다운로드된다

성공 기준:
- [ ] 내보내기 클릭 -> JSON 파일 다운로드
- [ ] 다운로드된 파일에 모든 카드 데이터 포함

### 14. 데이터 가져오기
[상황] 사용자가 JSON 파일을 가져오면
[동작] 기존 보드 데이터를 완전히 교체하여 복원한다

성공 기준:
- [ ] 유효한 JSON 파일 가져오기 -> 보드가 파일 내용으로 교체
- [ ] 잘못된 JSON 파일 가져오기 -> 에러 메시지 표시, 기존 데이터 유지

### 15. 데이터 영속성
[상황] 사용자가 카드를 추가/수정/이동한 후 페이지를 새로고침하면
[동작] 모든 보드 상태가 유지된다

성공 기준:
- [ ] 카드 추가 후 새로고침 -> 추가한 카드가 그대로 표시
- [ ] 칼럼 이동 후 새로고침 -> 이동한 칼럼에 카드가 표시

### 16. 우선순위 색상 구분
[상황] 카드에 우선순위가 설정되어 있으면
[동작] 우선순위별로 다른 색상이 표시된다

성공 기준:
- [ ] High 우선순위 -> 빨간색 계열 표시
- [ ] Medium 우선순위 -> 노란색 계열 표시
- [ ] Low 우선순위 -> 초록색 계열 표시

## 범위

### 포함
- 3개 고정 칼럼 (Todo, In Progress, Done) 칸반 보드
- 카드 CRUD (추가, 삭제, 인라인 제목 편집, 모달 상세 편집)
- 드래그&드롭 (칼럼 간 이동, 칼럼 내 순서 변경)
- 카드 속성 (제목, 우선순위, 태그, 마감일, 서브태스크)
- 검색 및 필터 (제목 검색, 우선순위 필터, 태그 필터)
- 다크모드 토글 및 저장
- JSON 내보내기/가져오기
- localStorage 기반 데이터 영속성

### 제외
- 칼럼 추가/삭제/이름 변경 — 요구사항에 3개 고정 칼럼으로 명시
- 사용자 인증/멀티유저 — 로컬 전용 앱 범위
- 카드 코멘트/첨부파일 — 요구사항에 포함되지 않음
- 서버 연동/API — localStorage 기반으로 명시
- 키보드 단축키 — 요구사항에 포함되지 않음
